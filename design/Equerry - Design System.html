Equerry
Design System · v1.0
Equerry Web · Field Guide direction

A calm system
for a private assistant.

Tone & mood. Editorial restraint over startup gloss: warm paper surfaces, a single trustworthy green, generous space, and humanist type. No gradients, no neon, no dark-pattern urgency — the craft is the trust signal. Equerry should feel like a well-made field manual that happens to live on your phone: quiet, legible, and honest about what it does. This page is the living reference — flip the theme, top-right, to see every token and component in light and dark.

Reading column settings

Long-form docs render at 17px / 1.75 on a ~70-character measure (--container-prose: 46rem). Links take the brand green; like this one. Inline code such as slot.chat gets a soft green wash chip rather than a hard box.

Muted lead/aside text steps down in contrast while holding the same measure, so the rhythm of the column never breaks.

Feature cards · lead with Capability-Slots

Capability-Slot system

Wire different jobs to different backends — each capability is a labelled socket you plug a model into.

chatollama · llama3.1
visionanthropic · claude

Privacy is the product

No accounts, no telemetry, no first-party data. Reads the screen via the sanctioned Assist API — and you confirm every send.

FOSS · GPL-3.0

On F-Droid and Google Play. Every fork stays open — openness is the privacy guarantee.

Callouts / admonitions
Note

Equerry registers as your system assistant. You can switch back to any other assistant at any time in Android settings.

Tip

Point the chat slot at a local Ollama model to keep everyday queries entirely on-device.

Caution

Cloud backends send your prompt to a third party. Equerry shows the destination and asks before every send — but the choice of provider is yours.

Danger

Removing the default-assistant role disables screen-reading entirely until it is re-granted.

Code blocks
~/.config/equerry.tomlcopy
# capability slots — bring your own backend
[slots]
chat      = "ollama://llama3.1"      # local, private
vision    = "anthropic://claude"     # screen-reading
summarize = "openrouter://auto"

[privacy]
confirm_before_send = true
telemetry           = "off"

Install & set up

Get Equerry running as your default assistant in three steps.

Install from F-Droid or Google Play, then open Equerry and grant the default-assistant role. Equerry only reads the screen when you explicitly invoke it through the Assist API.

Note

You confirm every send. Nothing leaves the device without a tap.

Next, wire your first chat slot to a backend in equerry.toml and you're ready to ask.

Tailwind v4 @theme

The copy-paste token file ships alongside this page as equerry-theme.css. Semantic colors map with @theme inline so utilities flip automatically under .dark — no duplicate classes.

equerry-theme.css · excerpt
@import "tailwindcss";
@plugin "@tailwindcss/typography";
@custom-variant dark (&:where(.dark, .dark *));

/* semantic vars flip on .dark */
:root {
  --bg:#F5F3EC; --surface:#FFFFFF; --text:#20211C; --muted:#736E60;
  --primary:#3A7D5C; --wash:#EAF1EA; --border:#E8E3D7;
}
.dark {
  --bg:#141611; --surface:#1C1F18; --text:#F1EFE7; --muted:#9C978A;
  --primary:#62B98C; --wash:#1B2A20; --border:#2D3128;
}

/* map → utilities (bg-bg, text-muted, bg-primary, …) */
@theme inline {
  --color-bg: var(--bg);
  --color-primary: var(--primary);
  --color-border: var(--border);
  --font-display: "Hanken Grotesk", sans-serif;
  --radius-lg: 14px;
  --container-prose: 46rem;
}

Full file includes @font-face blocks, the complete palette, type scale, radius/shadow/container tokens, and the prose mapping for @tailwindcss/typography.
